## 一、为什么大家都盯着"香港 VPS"

香港机房离大陆近，物理延迟天然就低；它不需要备案，ICP 流程绕开；它进出带宽都是国际线路，海外业务和国内访问能兼顾。对做小站、跑代理、挂机器人、做跨境电商、自建 Gitea 的人来说，香港 VPS 几乎是"懒人最优解"——不用折腾资质，开机即用，延迟比美国日本韩国都舒服。

## 二、香港 VPS 主流线路全梳理

香港到大陆，无非就这几种主流回程路径，搞清楚它们的脾气，你挑套餐就能对症下药。

**1. CN2 GIA（电信精品网）**

电信高端线路，走 AS4809，丢包低、晚高峰稳定。代价是贵，月付十几加元起步很正常。适合电信用户重度依赖场景，比如远程桌面、视频会议、对丢包敏感的代理。

**2. CN2 GT（电信普通 CN2）**

比 GIA 次一档，也是走 59.43 节点但带宽和优先级差一截，晚高峰容易抖动。一般作为"伪 CN2"宣传，价格会便宜不少。

**3. AS9929 / AS4837（联通精品与骨干）**

AS9929 是联通的 VIP 骨干，AS4837 是联通普通骨干。联通用户挑这两条基本不会出错，回程延迟通常 30-50ms。

**4. CMI（移动国际）**

中国移动的香港出口，223.120 开头的 IP 段。移动用户走 CMI 是最舒服的，沿海地区延迟甚至能压到 30ms 出头。

**5. BGP 直连（多线整合）**

不是单一线路，而是把电信、联通、移动三条直连都做进机房路由里，根据你的运营商自动匹配最佳路径。优点是三网都不差，缺点是三网都不算顶配，但胜在通用、价格亲民。

**6. 国际线路（NTT / Telstra / Cogent / HKIX）**

走国际骨干绕进香港，不针对大陆做特别优化。延迟普遍 150ms+，但带宽给得大方，500Mbps 起步很常见，价格还压得很低。适合做海外业务、跑大流量下载、不在意大陆访问体验的场景。

Cloudie 是 VMISS 在香港最早合作的机房，线路相对成熟。三网去程都是直连，回程电信会走 CN2，联通走 AS10099 接 AS4837，移动走 CMI。延迟大致在 30-50ms 区间，适合对延迟不极致敏感、但希望三网都过得去的用户。100Mbps 起步带宽做小站、跑 Telegram bot、做 API 转发都够用。
